# Outfit Prompt Test Runner - AI 사용 설명서
/venv환경을 활성화하고 사용하세요


## 개요

이 테스트 프로그램은 **서버 실행 없이** 복장 추출 프롬프트를 독립적으로 테스트하고, LLM 응답의 품질과 형식을 검증하는 도구입니다.

## 위치

```
customprompt/test_program/main.py
```

## 실행 방법

프로젝트 루트에서 실행:

```bash
cd customprompt/test_program
python main.py [옵션]
```

또는 프로젝트 루트에서 직접:

```bash
python customprompt/test_program/main.py [옵션]
```

## 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--prompt FILE` | 테스트할 프롬프트 파일명 (customprompt/ 폴더 내) | `haganai_extract_outfit_prompt.py` |
| `--real` | `workflow_backup/mode/outfit_mode/`의 실제 데이터 사용 | 샘플 데이터 |
| `--show` | 입력 데이터 상세 표시 | OFF |
| `--filter NAME` | 캐릭터 이름으로 테스트 필터링 (부분 매칭) | 전체 |
| `--model MODEL` | LLM 모델 오버라이드 (예: gpt-4.1, gemini-3-flash-preview) | config.json 설정 |
| `--raw` | LLM 호출 없이 프롬프트 빌드만 확인 | OFF |
| `--help` | 도움말 표시 | - |

## 실행 예시

### 1. 기본 테스트 (샘플 데이터 + GPT-4.1)
```bash
python main.py
```
- 내장된 4개 샘플 테스트 케이스 실행
- config.json의 LLM 설정 사용

### 2. 실제 데이터로 테스트
```bash
python main.py --real
```
- `workflow_backup/mode/outfit_mode/`의 data.json 파일에서 실제 캐릭터 데이터 로드

### 3. 특정 캐릭터만 테스트
```bash
python main.py --filter kobato
```

### 4. 프롬프트 빌드만 확인 (API 호출 없음)
```bash
python main.py --raw
```
- LLM 호출 없이 system/user 프롬프트가 어떻게 조합되는지 확인
- 프롬프트 수정 후 빠른 검증에 유용

### 5. 다른 모델로 테스트
```bash
python main.py --model gemini-3-flash-preview
```

### 6. 다른 프롬프트 파일 테스트
```bash
python main.py --prompt 복장추출_기본구조.py
```

## 출력 형식 검증 기준

LLM 응답은 다음 JSON 형식이어야 합니다:

```json
{
    "headgear": "머리 장식 설명 (모자, 헤어밴드, 장신구 등)",
    "clothing": "주 의상 설명 (드레스, 유니폼, 캐주얼 등)",
    "shoes": "신발 설명 (부츠, 힐, 스니커즈 등)",
    "accessories": "액세서리 설명 (보석, 가방, 벨트 등). 없으면 None",
    "memo": "추가 맥락이나 특이사항"
}
```

### PASS 조건
- 응답이 유효한 JSON
- 5개 필수 키(headgear, clothing, shoes, accessories, memo) 모두 포함

### FAIL 조건
- JSON 파싱 실패
- 필수 키 누락

## AI 프롬프트 개선 워크플로우

### 1단계: 현재 상태 파악
```bash
python main.py --show
```
결과를 확인하여 어떤 캐릭터가 FAIL인지 파악합니다.

### 2단계: 프롬프트 확인
```bash
python main.py --raw
```
system/user 프롬프트가 어떻게 조합되는지 확인합니다.

### 3단계: 프롬프트 수정
`customprompt/haganai_extract_outfit_prompt.py`의 다음 함수들을 수정합니다:
- `_build_system_prompt()`: 역할 정의와 출력 형식
- `_build_user_prompt()`: 입력 데이터 조합 방식
- `_get_char_identity()`: 캐릭터별 스타일 정보

### 4단계: 재테스트
```bash
python main.py --filter [문제 캐릭터]
```
특정 캐릭터만 빠르게 재테스트합니다.

### 5단계: 결과 비교
`results/` 폴더의 이전 결과와 새 결과를 비교하여 개선 여부를 확인합니다.

## 주요 함수 참조

### haganai_extract_outfit_prompt.py

| 함수 | 용도 |
|------|------|
| `run()` | 메인 진입점 - 서버에서 호출 (previous_result 인자 포함) |
| `_build_system_prompt()` | 시스템 프롬프트 생성 |
| `_build_user_prompt()` | 유저 프롬프트 생성 (previous_result 인자 포함) |
| `_get_char_identity()` | 캐릭터 스타일 정보 반환 |
| `clean_chat()` | 채팅 텍스트 정제 |
| `gpt2gemini()` | GPT messages → Gemini 형식 변환 (system → user 병합) |
| `gemini2gpt()` | Gemini messages → GPT 형식 변환 (system 분리) |
| `_validate_output()` | JSON 출력 검증 |
| `_try_recover_json()` | 손상된 JSON 복구 |
| `_fill_defaults()` | 누락된 키 기본값 채우기 |

### main.py (테스트)

| 함수 | 용도 |
|------|------|
| `run_all_tests()` | 전체 테스트 실행 |
| `run_single_test()` | 단일 테스트 실행 |
| `run_raw_test()` | LLM 없이 프롬프트 빌드만 테스트 |
| `validate_output()` | 출력 형식 검증 |
| `load_real_data()` | 실제 데이터 로드 |
| `load_prompt_module()` | 프롬프트 파일 동적 로드 |

## 사전 요구사항

1. **API 키**: `key/copilot.json` 파일이 존재해야 함 (LLM 호출 시)
2. **Python 패키지**: `httpx`, `aiohttp` 설치 필요
3. **프로젝트 구조**: `modes/llm_service.py`가 접근 가능해야 함

## 트러블슈팅

| 문제 | 해결 |
|------|------|
| `Copilot API 키가 없습니다` | `key/copilot.json` 파일 확인 |
| `모듈 import 실패` | 프로젝트 루트에서 실행 중인지 확인 |
| `실제 데이터 없음` | `--real` 시 `workflow_backup/mode/outfit_mode/`에 data.json 있는지 확인 |
| `타임아웃` | 네트워크 상태 확인, `--model`로 다른 모델 시도 |
